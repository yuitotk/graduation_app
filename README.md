## サービス概要（3行）

写真＋メモでアイデアを視覚的に整理し、AIが新しい発想を引き出す創作支援サービスです。
キャラの登場位置やストーリーの流れも確認でき、迷わない物語づくりを支援します。
“ひらめきから完成まで”を一つの場所で扱えて、見通しの良い制作環境を提供します。

## このサービスへの思い・作りたい理由

自分がストーリー作りや設定づくりを進める中で、 アイデアが散らかって整理しづらくなったり、 キャラの登場位置や役割が分からなくなることがよくありました。
一般的なメモアプリでは、 創作に必要な“アイデアの断片管理”や“ストーリー全体の見通し”を一つの場所で扱うことが難しく、 写真や画像を直感的に挿入できない点にも不便さを感じていました。 そのため、もっと創作に特化したツールが欲しいと強く思うようになりました。
こうした課題を自分自身の手で解消し、 ひらめきを逃さず整理し、ストーリーづくりへ自然につなげられる環境を作りたい。 さらに、キャラや設定の登場位置を見通しよく把握し、物語全体を視覚的に管理できる仕組みを備える。 そこにAIの発想補助を加えることで、ユーザーらしさがより鮮明に表れるアイデア生成を実現したい―― その思いが、このサービスを作りたいと考えた原点です。



## ユーザー層について
① ストーリー創作をする人（主に10〜30代）
* アイデア整理やプロット構築、物語全体を視覚的に管理に悩む人が多い
* 漫画家志望の中心である10~20代はスマホ中心で創作するユーザーも多く、ブラウザアプリとの相性が良い


## サービスの利用イメージ
ユーザーはひらめいたアイデアを写真＋メモとして投稿し、 カード形式でどんどん蓄積していくことができます。
創作が詰まったときは「ランダム×AI発想」ボタンを押すと、 登録したアイデアや辞書ワードを組み合わせてAIが“物語のタネ”を生成し、 新しい展開のヒントを得られます。
また、ストーリーを作る段階では、イベント（出来事）と キャラ・アイテムを紐づけて管理し、 “誰がどこで登場するか” を時系列で確認できます。 これにより、物語全体の流れを視覚的に把握しながら構築することができます。
この流れにより、 アイデア管理 → 発想補助 → プロット整合性チェックまでを ワンストップで行える創作環境 を提供します。


## ユーザーの獲得について
ストーリー制作やアイデア整理に悩む創作者は、 X（旧Twitter）、Pixiv、note などで悩みを共有する一方、 具体的な解決策となるツールは Google検索 を通じて探す傾向があります。 「アイデア 整理」「プロット 作れない」「キャラ 登場位置 わからない」といった検索ワードは、 創作者が実際に抱えやすい課題をそのまま表しています。
特に漫画家・小説家志望の10〜20代前半のユーザーは、 スマホのブラウザで検索して情報を集めることが多いため、 検索結果に表示されやすいタイトル・説明文を整えることが、 サービス認知の重要なポイントになります。
本サービスは、検索ユーザーのニーズに直結する次の機能を備えています。
* 写真＋メモで簡単にアイデアを整理できる機能
* ランダム×AI発想で新しい展開のヒントを得られる機能
* ストーリーの流れとキャラ・設定の登場位置を一瞬で把握できる機能
これらは、ユーザーが検索を通じて解決したいと感じている悩みと強く一致しており、 サービス内容自体が自然な集客動線となります。

そのため、このサービスでは
創作者が自分の制作ノウハウや使用ツールを積極的に共有しづらい
X（旧Twitter）などのSNSからの流入よりも、 主にGoogleからの検索流入（SEO）を中心としたユーザー獲得 を想定しており、 創作に行き詰まったユーザーが検索を通じて訪れ、 「これだ、自分が求めていたものだ」と感じてもらえる導線づくりを重視しています。



## サービスの差別化ポイント・推しポイント

1️⃣ ランダム×AIで「物語のタネ」生成
一般的なお題アプリやAI文章生成ツールは存在しますが、
* ランダムに選ばれたアイデアや辞書ワードを組み合わせる
* その組み合わせを AI が“創作向けの短い物語文”に変換する
* ワンクリックで何度でも生成できる
といった 「創作に特化した発想エンジン」 を一連の流れとして備えるサービスはほとんどありません。
Notionなどの汎用ツールにもAI機能はありますが、 ユーザー自身がプロンプトやページ構造を工夫しなければならず、 「ランダム発想 → AI文章生成 → アイデアとして保存」 までが自動でつながる仕組みは用意されていません。
本サービスでは、創作者が抱えがちな
* 発想が止まる
* 展開が浮かばない
* いつも同じパターンになってしまう
といった悩みに対し、 AIが“次の一手”となるヒントを即座に提示できる点 が大きな強みです。
創作の行き詰まりをピンポイントで解消できる、 本サービス独自の価値を持つ機能となっています。

2️⃣ ストーリー整合性チェック機能
キャラ・アイテム・設定が「どのイベントで登場するか」を 時系列とセットで確認できるサービスはほぼ存在しません。
一般的なメモアプリやノートツールでは、
* キャラや設定とイベントの紐付けが複雑になる
* どこで何が登場したかを一覧化できない
* 完成したストーリーの流れを視覚的に把握しづらい
という弱点があり、 ストーリー全体の見通しを保ちながら制作することが難しくなります。
本サービスでは、
* イベントを時系列で管理
* キャラ・アイテムをイベントに紐付け
* 絞り込みで「どの要素がどこに現れるか」を一瞬で表示
* 完成済みストーリーの構造も縦スクロールで見える化
できるため、
* キャラの出番不足
* 物語の矛盾
* 設定の抜け・重複
* 長編で全体が把握しづらい問題
を 視覚的に、迷わずチェックできる環境 を提供します。
この “ストーリー構造 × 登場要素 × 視覚化” を一つの画面で扱える点は、 他サービスにはほとんど見られず、本サービス独自の強力な差別化ポイントです。


## 機能候補

🟦 MVP

期間：3〜6週間
難易度：中級の低〜中

本サービスの核となる 「アイデア整理」と「AIによる発想補助」 を最低限の機能に絞り込み、短期間で利用可能な形にするため、以下の機能をMVPリリースとして実装します。

1. アイデア投稿・管理機能（メモ）
ユーザーがひらめきを素早く記録できることを最優先とした基本機能です。
* タイトル・メモ入力
* アイデアの新規投稿 / 編集 / 削除
* 各ユーザーごとのアイデア管理
👉 創作アイデアを視覚的に整理できる中核機能

2. 辞書×AI発想機能（簡易版）
登録されたアイデアや辞書ワードをもとに、“物語のタネ” を生成する創作補助機能です。
* 辞書単語からランダムで2つ選択
* AIがその組み合わせを解釈し、短い物語アイデアを生成
* 生成したアイデアを新規カードとして保存可能
👉 創作が行き詰まったときの突破口となる、サービス独自の発想エンジン

3. ユーザー登録・ログイン
ユーザーのアイデアがプライベートに管理されることを目的とした認証機能です。
* 新規登録
* ログイン / ログアウト
* 自分のアイデアだけが表示される仕組み
👉 創作データの保護に不可欠な基本要素

4. トップページ（簡易版）
ユーザーが迷わず使い始められる導線を用意します。
* サービス概要の簡易表示
* 「アイデア一覧」「新規投稿」「AI発想」への導線配置
👉 利用開始までのステップを最短化するUI



🟦 本リリースまでに作っていたい機能

6〜10週間（1.5〜2.5ヶ月）
本リリース全体の難易度は「中級の上〜上級の手前くらい

MVPで 「アイデア整理」と「AIによる発想補助」 を先にリリースしたあと、 本リリースでは当初のコンセプトである
アイデア整理 → 発想補助 → プロット整合性チェック
までを一つのサービス内で完結できるようにすることを目標に、 以下の機能を追加実装したいと考えています。

1. ストーリー時系列管理機能
物語全体の流れを見通しよく把握するための機能です。
* ストーリー内の「イベント（出来事）」を登録
* 各イベントに 順番（position） や章番号を設定
* タイトル / 簡単な説明 / メモを保存
* イベント一覧を 時系列順のリスト（タイムライン） として表示
👉 完成済みのストーリーも、 「どんな出来事が、どの順番で起きているか」を 一目で確認できる状態を目指します。

2. キャラ・アイテム・設定要素の管理
ストーリーに登場する要素を整理するための機能です。
* キャラクター
* 武器・アイテム
* 世界の設定・組織・場所 など
について、
* 名前
* 種類（キャラ／武器／アイテム／設定 など）
* 簡単な説明やメモ
といった情報を登録・編集できるようにします。
👉 「誰がいて、どんな要素があるのか」を一覧で把握できる状態にします。

3. イベント × 登場要素 の紐付け（多対多管理）
「どのイベントに、どのキャラ・アイテムが関わっているか」を 管理できるようにする機能です。
* 各イベントに対して、登場するキャラ・アイテムを複数紐づけ
* キャラやアイテムの詳細ページから 「登場しているイベント一覧」を確認できるようにする
👉 これにより、 「このキャラはどこで活躍しているか？」 「この武器はどのシーンで使われたか？」 がすぐに追えるようになります。

4. ストーリー整合性チェック用の絞り込み画面
キャラの出番不足や設定の抜けを確認しやすくするための画面です。
* 上部に「キャラ／武器／アイテム」などの絞り込みメニューを設置
* 特定のキャラを選ぶと、 そのキャラが登場するイベントだけを 時系列で一覧表示
* イベント番号（何番目か）も一緒に表示し、 「前回の出番からどれくらい間が空いているか」が把握しやすいUIにする
👉 これにより、
* キャラの出番が極端に少ない
* ある設定が途中から一切出てこない
* 物語の途中で要素が突然消える
といった問題を、完成しているストーリーでも視覚的に確認しやすくすることを目指します。

5. 辞書×AI発想機能の拡張
MVPで実装した「辞書ワード × 辞書ワード」モードに加えて、 より細かい発想支援モードを追加したいです。
* 辞書ワード × アイデア
    * アプリ内のキーワード（例：森／学園／近未来 など）と 自分のアイデアを組み合わせたタネ出し
* キャラ × アイデア
    * 特定のキャラを指定して、 「このキャラが活躍するイベント案」をAIに出してもらう
* 生成結果から、ワンクリックで新規アイデアカードを作成
👉 ストーリーがある程度進んでからでも、 「このキャラをもっと活躍させたい」 といったニーズに応えられる発想補助を目指します。

6. タグ・カテゴリ機能
アイデアやイベントが増えても整理しやすくするための機能です。
* アイデアカードにタグ（例：キャラ案／世界観／シーン案 など）を付与
* タグ別・カテゴリ別に一覧表示
* 将来的には、タグでAI発想の材料を絞り込めるようにすることも検討
👉 長期的に使っても「探せない」「埋もれる」を防ぐための整理機能です。

7. ストーリー／アイデアの簡易エクスポート
他ツールでの執筆や管理にも活用できるように、 以下のような形でのエクスポートを検討しています。
* ストーリーのイベント一覧を Markdown 形式で出力
* アイデアカードのタイトル＋メモをテキストとして出力
👉 小説本文やネーム作成ツールに持ち込むための 「橋渡し機能」として位置づけます。

8. レスポンシブ対応（スマホ最適化）
10〜20代の創作者はスマホ利用が中心であるため、最低限のモバイル最適化を実施します。
* カード表示のスマホ最適化
* ボタン配置の調整
👉 主要ユーザー層に合わせた基本的なUX向上

9. 画像アップロード機能
10. 外部ストレージ
11. お問い合わせ（入力→確認→完了）
12. パスワードリセット（申請／再設定）
13. 利用規約、プライバシーポリシー、
